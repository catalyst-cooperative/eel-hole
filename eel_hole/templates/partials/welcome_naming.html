Decoding Table Names

Intrepid PUDL spelunkers might notice we use a naming convention for our tables and columns. Knowing a little about it might help you better find what you are looking for.

{layer}_{source}__{asset_type}_{asset_name}

layer
is the level of processing of the table.
core
indicates the table is tidy or well-normalized [cg: add a good link of an actual beginner intro to tidy data]. Data in this layer should be cleaned, well-structured and minimally modified.
out
We recommend you start here. This indicates wide and complete tables suitable for users to perform analysis on. This means we've added back in a lot of the helpful TODO
_core
This is an intermediate table - meaning this table has not been fully cleaned. We are probably publishing it because we want it out in the world so you can use it but we don't love its level of cleanliness.
source
is an abbreviation of the original source of the data. For example, eia860, ferc1 and epacems. See list of sources here [link to jump back to search by data source].
asset_type
describes how the asset is modeled. [include pared down details from this longer list of options: assn, changelog, codes, entity, scd, timeseries]
asset_name
should describe the entity, categorical code type, or measurement of the asset. Sometimes this also includes information about the original source like how FERC Form 1 tables include the table's schedule number.

For more information on how we name tables and columns, see PUDL Naming Conventions.
